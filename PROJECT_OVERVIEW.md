# SAMI: Social Integrity System (FactShield)

## Project Overview
SAMI (Social Integrity System) is an advanced AI-powered fact-checking and propaganda detection platform. It combines traditional machine learning with modern Large Language Models (LLMs) and Retrieval-Augmented Generation (RAG) to provide comprehensive analysis of news claims, images, and videos.

---

## Backend Architecture

### Core Technologies
- **Framework:** FastAPI
- **Language:** Python
- **Database:** SQLite (Task tracking), FAISS (Vector database for RAG)
- **Models:** 
  - **Classification:** XLM-Roberta (Local)
  - **Reasoning:** Llama 3.3 (via Groq)
  - **Transcription:** Faster-Whisper
  - **OCR:** Pytesseract & OpenCV
  - **RAG:** FAISS with Cross-Encoder Reranking

### Components & Logic
1. **Verification Pipeline:**
   - **Classification:** Categorizes text based on veracity and toxicity using a local multi-label model.
   - **Live Search & Scraping:** If model confidence is low, the system automatically searches the web for live news and scrapes relevant articles to gather fresh evidence.
   - **RAG (Retrieval-Augmented Generation):** Retrieves relevant facts from a local FAISS vector index to provide ground-truth evidence.
   - **LLM Reasoning:** Uses Llama 3.3 to analyze the claim alongside retrieved evidence, generating detailed reasoning and historical context.
   - **Explainable AI (XAI):** Generates word-level attribution scores to explain why the AI made a specific prediction (Mechanistic Interpretability).

2. **Multimodal Analysis:**
   - **Text:** Direct analysis of text claims.
   - **Image:** Uses OCR (Tesseract) to extract text from images before processing.
   - **Video:** Performs OCR on video frames and uses Faster-Whisper to transcribe spoken audio, combining both for a full-context analysis.

3. **Background Processing:**
   - Heavy ML tasks are handled as background processes using FastAPI's `BackgroundTasks`, with status tracking in an SQLite database.

4. **Trending & Intelligence Feed:**
   - Provides live feeds of "trending propaganda" and ingested fact metadata to show the system's continuous learning.

## Frontend Architecture

### Core Technologies
- **Modern UI Stack:** React 18 (via CDN), Tailwind CSS (via CDN)
- **Communication:** Fetch API with dynamic polling for BackgroundTasks.
- **Visualization:** Vis-Network (Evidence Graphs), Lucide (Icons), Mechanistic Interpretability Heatmaps (XAI).

### Components
1. **Verification Command Center:**
   - Unified intake for text, images, and video files.
   - Real-time polling and status tracking for heavy ML tasks.

2. **Intelligence Feed & Trending Dashboard:**
   - **Intelligence Feed:** Live monitoring of news and social alerts.
   - **Trending Analytics:** Real-time propaganda cycles and narrative tracking.

3. **Results & Interpretability Overlay:**
   - **Veracity & Toxicity Badges:** High-visibility classification results.
   - **Neural Reasoning:** Detailed explanation generated by Llama 3.3.
   - **Evidence Graph:** Interactive visualization of fact-checked connections.
   - **XAI Heatmap:** Token-level attribution showing model decision-making.

4. **Evidence Archive & System Health:**
   - Historical database of verified facts from local memory.
   - Live system metrics (Neural load, vector engine state, model status).

---

## Project Logic Flow
1. **Input:** User submits text, image, or video.
2. **Pre-processing:** Media is converted to text (OCR/Transcription).
3. **Detection:** Local model performs initial classification.
4. **Augmentation:** System fetches external evidence (Search/RAG).
5. **Synthesis:** LLM creates a human-readable explanation based on evidence.
6. **Explanation:** XAI generates interpretability data.
7. **Delivery:** Frontend polls and displays the final multifaceted result.
